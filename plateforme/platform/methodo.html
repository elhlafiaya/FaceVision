Décision Ternaire
Feature Engineering : le choix clé
Pourquoi |e₁−e₂| ∈ ℝ⁵¹² et pas la distance cosine seule
❌ Approche Naïve
Distance cosine seule
1 seule valeur scalaire — perte d'information massive. Le SVM apprend à seuiller une dimension unique, équivalent à la baseline. Aucune information sur quelles composantes de l'embedding diffèrent.
✅ Approche Retenue
|e₁−e₂| ∈ ℝ⁵¹²
512 valeurs — chaque dimension i capture l'écart sur la i-ème composante discriminante FaceNet. Le SVM apprend quelles dimensions sont importantes, lesquelles sont du bruit. Invariant à l'ordre des paires.
X = |e₁ − e₂| ∈ ℝ⁵¹²    (différence absolue, invariante à l'ordre)
e_i ∈ ℝ⁵¹² normalisé L2 : ||e_i||₂ = 1    (FaceNet InceptionResnetV1 VGGFace2)
Xᵢ = |e₁ᵢ − e₂ᵢ|    ∀ i ∈ {1, ..., 512}
→ StandardScaler(X) → SVM RBF avec kernel K(x,x') = exp(-γ||x−x'||²)
Décision Ternaire
Du binaire rigide au niveau de confiance explicite
✅
ACCEPTÉ
P(genuine) ≥ Thigh = seuil + 0.15
Décision automatique positive. Le niveau de confiance dépasse largement le seuil EER. Genuine attendu avec haute certitude.
⚠️
INCERTAIN
Tlow ≤ P(genuine) < Thigh
Alerte explicite — décision suspendue. Zone de chevauchement réel des distributions. Vérification secondaire recommandée (second facteur, contrôle humain).
❌
REJETÉ
P(genuine) < Tlow = seuil − 0.15
Décision automatique négative. Score bien en dessous du seuil EER. Impostor attendu avec haute certitude.
Tlow = EER_threshold − 0.15    (zone de rejet clair)
Thigh = EER_threshold + 0.15    (zone d'acceptation claire)
Zone INCERTAIN : Tlow ≤ P(genuine) < Thigh    (coïncide avec le chevauchement statistique)
Justification (Cellule 15 du notebook) : La zone d'incertitude n'est pas arbitraire — elle est positionnée exactement là où les distributions genuine/impostor se chevauchent encore. Sur ORL : 2.6% de paires INCERTAIN (46/1800) ; sur Yale : 0.4% (3/770). Les erreurs (FA/FR) du seuil fixe qui tombent dans cette zone deviennent des alertes explicites plutôt que des décisions silencieuses.
Évaluation — Standards ISO/IEC 19795-1
Au-delà d'AUC/EER — mesurer la qualité de la confiance elle-même
AUC ROC
Aire sous la courbe ROC — capacité discriminante globale du classifieur.
EER (Equal Error Rate)
Taux d'erreur où FAR = FRR. Métrique standard biométrie. Seuil EER = point d'équilibre.
FAR / FRR
Faux Acceptés (imposteurs acceptés) / Faux Rejetés (genuines rejetés) au seuil EER.
DET Curve
Detection Error Tradeoff — échelle normale : linéarise les courbes FAR vs FRR pour comparer les systèmes.
d' (d-prime)
Indice de discriminabilité : séparation en σ entre les distributions genuine/impostor. d' > 4.9 mesuré.
ECE (Expected Calibration Error)
Écart moyen pondéré confiance ↔ fréquence réelle. Valide que P(genuine)=0.7 → ~70% genuine.
Brier Score
Erreur quadratique moyenne entre P(genuine) et label réel. 0 = parfait. Mesure calibration globale.
5-Fold CV Stratifiée
Cross-validation stratifiée sur les paires d'entraînement. Rapporte AUC mean ± std pour estimer la variance.
Reliability Diagram
Diagramme de fiabilité (Fig A) : confiance prédite vs fréquence observée bin par bin. Vérifie la calibration Platt visuellement.
